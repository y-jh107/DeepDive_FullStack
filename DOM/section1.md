### DOM
- Document Object Model,
- 핵심은 사용자와의 상호작용
- 동적으로 콘텐츠 표현
- 다양한 기술과 연동되며 자바스크립트로 융합

### 콘텐츠 표현 플로우

- html 파일의 마크업을 html로 파싱 랜더링 
- 콘텐츠 표현을 위해 **Object** 모델로 변환(DOM, CSSOM, AOM)

### 코드 작성 기준

- 함수 이름에 시맨틱을 부여하면 함수 블록의 코드를 읽지 않아도 함수의 기능을 짐작할 수 있음
- 이처럼 변수, 함수 이름에 시맨틱을 부여하면 코드가 시맨틱으로 연결된 흐름이 되고 코드가 흐름을 가진 하나의 시나리오가 됨
- 코드 단위로 시맨틱을 부여하는 것이 아니라 앞뒤가 연결된 시나리오가 되도록 시맨틱을 부여
- 뉘앙스(목적)를 하나만 갖도록 해야 시나리오가 단순해지며 쉽게 목적에 맞는 시나리오를 만들 수 있음

### 뉘앙스 단순화 방법

- 시맨틱 그룹: 시맨틱에 속하는 함수를 오브젝트에 작성하면 오브젝트는 함수 묶음이 됨
- 함수는 시맨틱과 뉘앙스를 갖고 있으므로 오브젝트는 시맨틱 묶음이 되며 함수를 대표하는 목적(뉘앙스)을 갖게 됨

### DOM 접근 키워드

- 인터페이스, **Property**, 메소드의 목적을 파악
- 코드 중심이 아니라 DOM 전반의 감각을 만드는데 중점을 두고 시나리오를 시스템 구상, 설계, 개발에 적용

### DOM 인터페이스

- document의 타입은 object
- document는 Document 인터페이스를 오브젝트로 생성한 것

### Host Environment

- JavaScript 엔진이 실행되는 환경
- JS 자체가 제공하지 않는 기능들을 외부에서 붙여주는 실행공간(ex - 웹 브라우저)

### 함수 실행 단계

- 호출 전 - 실행을 위한 기반 환경 설정
- 호출 후 - 함수를 호출하여 함수 실행

### **Object**

- 오브젝트는 0개 또는 그 이상의 프로퍼티들의 collection
- 각각이 프로퍼티가 어떻게 사용될 수 있는지 결정하는 속성이 있음

### **Property**

- 다른 **Object**, 원시 타입, 함수를 담을 수 있는 컨테이너
- 자바스크립트는 object를 기반으로 하며 object를 구성하는 것이 프로퍼티이므로 자바스크립트의 형태와 구조의 바탕은 object와 **Property**

### 함수, 메소드, Global Object

- 함수는 호출가능한 **Object**
- 오브젝트의 프로퍼티로 존재하면 메소드
- **Global Object**도 **Object**이며 **Object** 이름이 Global, **Property**와 함수가 Window 오브젝트에 설정되는 것

### 실행 Context, Scop chain

- Context는 하나의 덩어리 형태
- 실행 Context: 함수가 호출되면 실행 context를 만들고 함수를 하나의 context로 실행
- 실행 context에 호출된 함수에 작성된 변수 파라미터 등의 모든 것을 {key: value} 형태로 설정하고 이 안에서 함수 블록의 코드 실행 
- Scope: 식별자 해결을 위한 범위
- Property 이름을 식별하는 순서, 방법:
    1. 함수 안에서 먼저
    2. 존재하지 않으면 함수 밖에서
    3. 없으면 글로벌 오브젝트까지 올라가면서
    4. 올라가는 단계 하나하나가 스코프
- 함수를 기준으로 함수 밖을 클로저라고 함

### HTML 정리

- 엘리먼트: <>(시작태그), </>(끝태그), id는 속성 이름, "텍스트"는 Content, 범위는 시작태그에서 끝태그까지의 전체
- 태그: 태그 이름 === 엘리먼트 이름, DOM의 처리 기준은 엘리먼트(시작태그~끝태그)
- HTML: 애플리케이션 중심, 콘텐츠를 표현하는 하나의 매체

### HTML의 구조, 구성, 아웃라인

- HTML은 표현이 아니라 콘텐츠를 제공하기 위한것
    1. 표현은 HTML이 아니라 CSS로
    2. 표현 중심으로 접근하면 \<div>로 계층 구조의 마크업을 만들게 됨
- 아웃라인: 콘텐츠를 의미적으로 구분하기 위한 구조(구성)를 정형화, 명시화

### \<section>와 아웃라인

- section 엘리먼트: a generic section(책의 절 또는 항 단위의 콘텐츠) of a document or application
- section 아웃라인: 알고리즘은 section을 들여쓰기 개념으로 바라봄, 메인 콘텐츠 흐름에 참여
- section 아웃라인 참고사항: section에 heading 작성 가능, section이 콘텐츠 흐름의 연결이지만 \<h1> 작성 가능, MDN 문서에서 \<h1>~\<h6>로 사용 권장

### HTML 스펙의 엘리먼트

- title 엘리먼트: 
    - Categories - 특성이 비슷한 엘리먼트의 카테고리
    - Contexts in which this element can be used - 엘리먼트가 엘리먼트의 context 위치에서 Context로 사용되는 엘리먼트 또는 카테고리
    - Content model - 엘리먼트의 Content 위치에 작성할 수 있는 것
    - Tag omission in text/html - 태그 작성을 생략하는 경우 \<head>, \<body>의 시작 태그 생략 가능
    - Content attributes - 엘리먼트에 작성할 수 있는 속성
    - IDL 속성 - DOM의 프로퍼티, 메소드로 엘리먼트 오브젝트를 만들고 자바스크립트의 프로퍼티 키로 속성값을 구함
    - DOM interface - 엘리먼트 오브젝트를 생성하여 자바스크립트로 처리하는 것

### 요구 분석, 시나리오

- 시나리오: 요구사항을 처리하는 시나리오를 만들고 시나리오를 따라가면서 프로그램을 개발, 할 것과 실행 순서가 있으며 프로그램 코드도 같음
- 시나리오 작성 형태: 주어 + 동사 + 목적어
    - 무엇을 생성한다. 
    - 무엇을 실행한다.
    - 문장 단위의 시나리오가 연결되어야 함
- 요구 분석: 시나리오를 연결하는 것
- 이벤트는 다음 프로세스가 실행되는 원인 제공, 프로세스를 실행하면 이벤트가 발생
- 사용자 중심 모델 시나리오: 시스템 처리 중심 모델 시나리오를 별도로 작성

### 실무 기준

- 프로그램 개발: 요구사항을 구현하는 것, 요구사항에 최적화된 프로그램을 개발하기 위해서는 최적의 오브젝트, 메소드, 프로퍼티를 사용해야 함
